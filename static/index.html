BLF 解析 & CAN 信号解码原理

本程序如何将 BLF 二进制文件转化为可视化 CAN 信号波形

BLF 文件格式简介

BLF（Binary Logging Format）是 Vector Informatik 定义的二进制日志格式，用于 CANoe / CANalyzer 记录 CAN / CAN FD 总线数据。文件由文件头和一系列对象（Object）顺序组成，每个对象自带描述头，支持流式读写。

文件头（File Header）

signature4 字节，固定为 "LOBJ"（0x4C4F424A）
header_sizeDWORD，文件头大小（通常 144 字节）
api_versionDWORD，API 版本号
platformDWORD，创建平台（1=32-bit, 2=64-bit）
creation_flagsDWORD，创建标志（含压缩方式）
measurement_startSYSTEMTIME，测量开始时间
last_object_timestampSYSTEMTIME，最后对象时间戳

对象头（Object Header）

每个对象均以 VBLObjectHeader 开头：

mSignatureDWORD，对象签名 BL_OBJ_SIGNATURE
mHeaderSizeWORD，头大小（16 或 32 字节）
mHeaderVersionWORD，头版本（1 或 2）
mObjectSizeDWORD，整个对象大小（字节）
mObjectTypeDWORD，对象类型（见下方）
mObjectTimeStampULONGLONG，时间戳（10μs 或 1ns）

CAN 报文对象（VBLCANMessage2）

类型 BL_OBJ_TYPE_CAN_MESSAGE2，CAN 2.0 数据帧/远程帧：

mHeader对象头
mChannelWORD，CAN 通道号
mFlagsBYTE，标志位（DIR / RTR / WU / NERR）
mDLCBYTE，数据长度（最大 8）
mIDDWORD，帧 ID
mData[8]BYTE[8]，CAN 数据
mFrameLengthDWORD，帧时长（ns）

CAN FD 报文对象（VBLCANFDMessage64）

类型 BL_OBJ_TYPE_CAN_FD_MESSAGE_64，CAN FD 帧：

mHeader对象头
mChannelBYTE，CAN 通道号
mDLCBYTE，DLC 编码（0-15，解码见下表）
mValidDataBytesBYTE，有效数据字节数
mIDDWORD，帧 ID
mData[64]BYTE[64]，CAN FD 数据
mFlagsDWORD，含 FDF / BRS / ESI 等标志

CAN FD DLC 解码

DLC 值到实际数据长度的映射：

DLC0-89101112131415
字节数=DLC12162024324864

序列化 / 反序列化流程

1 写入：按 File Header → Object 1 → Object 2 → … 顺序追加，每个对象含自描述头
2 读取：先解析 File Header，再逐个读取 Object，根据 mObjectType 分发解码
3 压缩：对象可使用 zlib / deflate 压缩存储，由 creation_flags 控制
4 字节序：所有字段均为小端序（Little-Endian）

参考：Vector Informatik《CAN and General BLF Logging Format Specification》v1.45 (2018-07-20)
